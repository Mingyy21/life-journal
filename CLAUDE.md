# Claude Code 协作规则

## 项目信息
- **名称**：人生手记 (My Life Journal)
- **路径**：D:\总文件夹\项目\人生手记
- **阶段**：MVP v0.1 开发中

## 核心规则
1. **始终使用中文**进行回答和交流
2. 每次工具调用前，用**中文简短播报**：`[标签] 对象/说明`
3. 记忆达上限时，**remit压缩记忆**后再继续
4. 文件写入后**必须验证**（用Bash `ls -la` 或 Read读取）
5. **每次只串行写入1个文件**，写完验证成功后再写下一个

## 工作流程
每条开发任务按以下顺序执行：
1. **问第一性原理** — 这个功能对MVP是否必要？
2. **搜索参考方案** — 网上找3-5个最新实现
3. **写方案展示** — 给用户确认
4. **确认后执行** — 不可跳过确认
5. **验证结果** — 确保文件落盘

## 第一性原理
**记录是一切的上游**。MVP只解决一个问题：让用户用最低的摩擦完成记录。

## 项目关键文件
- **数据层**：`src/lib/db.ts` — Dexie.js数据库Schema + CRUD
- **AI分析**：`src/lib/ai.ts` — DeepSeek API调用 + 模拟分析
- **加密预留**：`src/lib/crypto.ts` — Web Crypto API（v0.2启用）
- **类型定义**：`src/types/index.ts` — 所有TS类型
- **全局布局**：`src/components/Layout.tsx` — Header + 底部导航
- **首页**：`src/app/page.tsx` — 写作入口 + 筛选 + 列表
- **日记详情**：`src/app/diary/[id]/page.tsx` — 详情 + AI分析

## 记忆模型 (v0.3+ 远景)
三层五机制：
- L1 情节记录层 → L2 认知模式层 → L3 生命叙事层
- 激活传播 + 情绪标记巩固 + 去重演化 + 优先级动态调整 + 跨周期审视
