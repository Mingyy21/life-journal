# 项目总览

## 第一性原理

**记录是一切的上游。如果没有高质量的记录，再聪明的AI也无事可做。**

MVP要解决的唯一问题：让用户用最低的摩擦完成记录，同时保证记录的结构足够好，好到将来AI能在上面工作。

## 项目要解决的两个核心问题

### 问题一：人类不善于看到自己的变化
你每天都在变，但你感知不到。三个月前的你和现在的你对同一件事的看法可能完全不同，但你自己很难对比。这个项目最大的价值，是让你看到"过去的自己→现在的自己"之间的那条线。技术上表现为**记忆对比+认知轨迹追踪**。

### 问题二：情绪是短暂的数据，认知才是沉淀的资产
一个人写了三年日记，积攒了几百条情绪记录，如果不做提炼，这些数据毫无意义。这个项目要做的事，是把短命的"感受数据"提炼成可积累的"认知资产"——你对友情的理解、你对愤怒的应对模式、你在压力下的思维习惯——这些东西会越来越精确，成为你的人生操作系统。

**让每次痛苦都变成可复用的智慧，让每次快乐都变成可追溯的根源。**

## MVP边界 (v0.1)

### ✅ 做
- 日记式记录（自由书写，无需模板）
- 课题标签系统：人际(友情/亲情/爱情)、人生(工作/学习)、自我(情绪/认知/健身)
- 按时间浏览日记列表（日期分组）
- 按标签筛选日记（课题→细化标签联动筛选）
- AI分析按钮（手动触发，非自动）：情绪标注(VAD分数)、课题关联、简短反馈
- 分析卡片展示：VAD三维指标 + Plutchik情绪分布条 + 认知阶段 + AI洞察 + 反馈 + 引导问题
- PWA基础：手机浏览器可访问、添加到主屏幕、离线缓存
- 本地IndexedDB存储（不依赖网络，数据在浏览器中）
- 不配置API Key也能使用（自动降级为模拟分析）

### ❌ 不做（但数据结构预留扩展口）
- 对话式记录（情绪消解模式）→ v0.2
- 多智能体系统（对话/课题分析/周期复盘）→ v0.3+
- 记忆优先级系统 → v0.3+
- 知识图谱 → v0.3+
- 云同步/加密 → v0.3+
- 语音输入 → v0.3+
- 智能提醒/置顶 → v0.2

## 远景功能（产品想法池）

用户提出但不在MVP中的想法，记录在 `docs/产品想法池.md` 中：
- **两种记录模式**：情绪消解式（AI对话引导）+ 独立记录式（自由书写）
- **三个智能体**：对话Agent、课题分析Agent、周期复盘Agent（周/月/年）
- **三层记忆模型**：情节记录层 → 认知模式层 → 生命叙事层
- **记忆优先级**：高频记录快速调取，低频记录深埋但可检索
- **跨课题模式识别**：发现你在不同课题下重复出现的思维/行为模式
- **进度可视化**：每个课题的消化进度（雷达图）
