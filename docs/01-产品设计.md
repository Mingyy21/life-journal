# 产品设计

## 用户场景

1. **日常记录**：用户在一天结束或某个触动的时刻打开App，写下当下的经历、感受和想法
2. **情绪波动**：情绪上来的时候，打开App快速记录，不给AI分析也不打标签，存起来再说
3. **回顾筛选**：积累一定量后，按课题标签查看某个领域的完整记录（比如只看"友情"相关的所有日记）
4. **深度分析**：对某篇特别在意的日记点"AI分析"，获得多维度的情绪洞察和个性化反馈
5. **手机写日记**：晚上躺在床上用手机写，早上在电脑前也可以继续写

## 交互流程

```
打开App → 看到日记列表（按日期分组，最新在上）
    ↓
点击顶部"+"按钮或空白区域 → 弹出编辑器
    ↓
写标题(可选) + 写正文 → 选课题标签(可多选) → 选细化标签(可选) → 点"保存"
    ↓
收起编辑器，新日记出现在列表顶部
    ↓
点进某篇日记 → 查看完整内容 → 点"AI分析"按钮
    ↓
看到分析卡片：情绪分布条 + VAD三维指标 + 认知阶段 + AI洞察 + 反馈 + 引导问题
```

### 筛选流程
```
在首页标签筛选区：
  点"全部" → 显示所有日记
  点"友情" → 只显示友情相关日记，展开细化标签
  点细化标签 → 进一步筛选
  再次点已选标签 → 取消筛选
```

## 页面结构（3个页面）

### 1. 首页（/）
- **写作入口区**：一个大面积的可点击空白卡片，写着"记录今天的想法..."
- **标签筛选区**：课题标签横排（全部 + 8个课题按钮），选中后展开细化标签
- **日记列表区**：按日期分组，每篇日记显示标题、内容预览(120字)、课题标签色块、时间、是否有AI分析(Brain图标)

### 2. 日记详情页（/diary/[id]）
- **返回按钮**
- **日记内容**：标题、日期时间、字数、课题标签、正文
- **AI分析区**：
  - 未分析：显示"✨ AI分析这篇日记"按钮
  - 分析中：加载动画
  - 已分析：展示分析卡片（可展开/收起）

### 3. 设置页（/settings，后续版本）
- API Key配置
- 课题管理（增删改）
- 数据导出
- 关于

## 5个参考产品分析

### 1. Apple Journal（苹果官方日记App）
- **特色**：极简写作入口，大卡片式布局，按日期可滚动
- **可用**：写作引导的简洁性、日期分组的视觉舒适度
- **不足**：无AI分析，标签系统简单

### 2. Day One（老牌跨平台日记App）
- **特色**：日历视图 + 多条时间线，Markdown编辑器，定位/天气自动记录
- **可用**：按时间组织的阅读体验，多维度元数据
- **不足**：付费订阅，AI功能有限

### 3. Reflectly（AI日记App）
- **特色**：AI对话式引导写日记，情绪色块可视化，个性化问题模板
- **可用**：情绪可视化的色彩设计，AI引导的对话感
- **不足**：过于依赖模板，不适合自由书写

### 4. Five Minute Journal（结构化日记）
- **特色**：早晚固定格式，正面心理学导向，简单到极致
- **可用**：降低写作门槛的引导设计思路
- **不足**：模板固定，无法自由扩展

### 5. 墨记（国内极简日记App）
- **特色**：极简中文书写体验，标签系统清晰，隐私保护
- **可用**：中文排版和字体选择，标签管理方式
- **不足**：无AI分析，仅本地使用

## 设计差异化

| | 现有产品 | 人生手记 |
|---|---|---|
| 写作方式 | 固定模板或纯自由 | 自由书写 + 可选标签引导 |
| AI角色 | 对话引导或简单总结 | 多维情绪分析 + 认知成长追踪 |
| 数据归属 | 云端（你管不了） | 本地优先（IndexedDB，数据在你手中） |
| 标签系统 | 单一标签 | 双层标签（课题→细化标签） |
| 长期价值 | 写日记本身 | 积累认知资产，追踪个人成长 |

## 移动端vs桌面端

- **手机**：书写入口最突出，列表查看为主，操作简化
- **电脑**：键盘输入体验更好，适合深度书写和批量回顾
- **PWA**：一套代码，自适应布局，无需开发两套
