## 数据流

```
用户写日记 → DiaryEditor组件(状态管理)
                ↓
          useDiary.addDiary()
                ↓
           db.diaries.add() → IndexedDB(本地持久化)
                ↓
          loadData() → 刷新列表显示

用户点"AI分析" → diary/[id]/page.tsx
                ↓
          POST /api/analyze { title, content }
                ↓
          analyzeDiary() → DeepSeek API
                ↓
          返回分析结果 → 存入 analysisResults 表
                ↓
          AnalysisCard组件渲染
```

## API设计

### POST /api/analyze
- **描述**：对一篇日记进行情绪和认知分析
- **Runtime**：Edge
- **输入**：`{ title: string, content: string }`
- **输出**：
```json
{
  "vadScore": { "valence": -0.5, "arousal": 0.7, "dominance": 0.3 },
  "emotionLabels": { "joy": 0.1, "sadness": 0.7, "anger": 0.4, ... },
  "primaryEmotion": "悲伤",
  "intensity": 0.65,
  "cognitiveStage": "觉察",
  "topics": ["友情", "情绪"],
  "tags": ["争吵", "失望"],
  "insight": "你在友情冲突中倾向于先内化情绪...",
  "feedback": "我能感受到你的失望...",
  "followUpQuestion": "如果用一个词形容这段友情的核心价值，会是什么？"
}
```
- **错误处理**：400(内容太短) / 500(API错误，降级为模拟分析)

## 离线策略（PWA）

| 资源类型 | 策略 | 说明 |
|---------|------|------|
| 页面HTML | NetworkFirst | 优先网络，失败时用缓存 |
| JS/CSS构建产物 | CacheFirst | 带hash的文件名，永不改变 |
| 字体/图标 | CacheFirst | 静态资源长期缓存 |
| API响应 | NetworkFirst | 分析结果缓存到IndexedDB |
| 日记数据 | IndexedDB | 完全不依赖网络 |

## 安全设计

- **API Key**：仅存在于 `.env.local`，通过服务端API Route调用，不暴露给客户端
- **HTTPS**：Vercel自动提供，所有数据传输加密
- **XSS防护**：React默认转义 + 内容安全策略(CSP)
- **无用户认证**：MVP阶段纯本地应用，不需要登录注册
- **加密预留**：`src/lib/crypto.ts` 已实现AES-256-GCM + PBKDF2，v0.2启用
