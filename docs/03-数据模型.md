# 数据模型

## 数据库

- **引擎**：IndexedDB（通过Dexie.js封装）
- **数据库名**：`LifeJournalDB`
- **版本**：v1
- **特点**：浏览器本地存储，无需网络，隐私安全

## 四张核心表

### 1. diaries（日记主表）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✅ | UUID，主键 |
| title | string | ✅ | 日记标题 |
| content | string | ✅ | 日记正文 |
| topics | string[] | ✅ | 关联课题ID列表 |
| tags | string[] | ✅ | 关联标签ID列表 |
| analysisId | string \| null | - | 关联分析结果ID |
| hasAnalysis | boolean | ✅ | 是否已有AI分析 |
| wordCount | number | ✅ | 正文字数 |
| createdAt | Date | ✅ | 创建时间 |
| updatedAt | Date | ✅ | 最后更新时间 |

**索引**：id, createdAt, updatedAt, *topics (多值索引), *tags (多值索引), hasAnalysis

### 2. tags（标签表）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✅ | UUID，主键 |
| name | string | ✅ | 标签名称 |
| topicId | string | ✅ | 所属课题ID |
| color | string | ✅ | 前端展示颜色 (hex) |
| diaryCount | number | ✅ | 关联日记数量 |
| createdAt | Date | ✅ | 创建时间 |

**索引**：id, topicId, name

### 3. topics（课题表）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✅ | 固定ID（如 topic-friendship） |
| name | string | ✅ | 课题名称 |
| category | "人际" \| "人生" \| "自我" | ✅ | 所属大类 |
| color | string | ✅ | 课题颜色 (hex) |
| icon | string | ✅ | Lucide图标名 |
| description | string | ✅ | 课题描述 |
| progress | number | ✅ | 消化进度 0-100（v0.2+使用） |
| diaryCount | number | ✅ | 关联日记数量 |
| lastReviewedAt | Date \| null | - | 上次复盘时间 |
| createdAt | Date | ✅ | 创建时间 |

**索引**：id, category, name

### 4. analysisResults（AI分析结果缓存表）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | ✅ | UUID，主键 |
| diaryId | string | ✅ | 关联日记ID |
| vadScore.valence | number | ✅ | 愉悦度 -1 ~ 1 |
| vadScore.arousal | number | ✅ | 唤醒度 0 ~ 1 |
| vadScore.dominance | number | ✅ | 支配度 0 ~ 1 |
| emotionLabels.joy | number | ✅ | 喜悦 0-1 |
| emotionLabels.trust | number | ✅ | 信任 0-1 |
| emotionLabels.fear | number | ✅ | 恐惧 0-1 |
| emotionLabels.surprise | number | ✅ | 惊讶 0-1 |
| emotionLabels.sadness | number | ✅ | 悲伤 0-1 |
| emotionLabels.disgust | number | ✅ | 厌恶 0-1 |
| emotionLabels.anger | number | ✅ | 愤怒 0-1 |
| emotionLabels.anticipation | number | ✅ | 期待 0-1 |
| primaryEmotion | string | ✅ | 主要情绪名称 |
| intensity | number | ✅ | 情绪综合强度 0-1 |
| cognitiveStage | string | ✅ | 认知阶段：觉察/接纳/理解/重构/行动 |
| topics | string[] | ✅ | AI判断的关联课题 |
| tags | string[] | ✅ | AI建议的细化标签 |
| insight | string | ✅ | AI洞察（1-2句） |
| feedback | string | ✅ | AI反馈（2-3句） |
| followUpQuestion | string | ✅ | 引导思考问题 |
| createdAt | Date | ✅ | 分析时间 |

**索引**：id, diaryId, createdAt

## 数据关系图

```
topics (1) ─────────< (N) tags
  │
  │ (多对多)
  │
  ├──────────< diaries >──────────┐
  │                                │
  │                           (1对1)
  │                                │
  │                     analysisResults
  │
  └── 一个日记可关联多个课题
      一个课题下有多篇日记
```

## 数据迁移策略

- Dexie.js schema版本管理
- 新增字段设默认值
- v0.1 → v0.2 需增加记忆模型相关表（新增version(2)）
- 不删除旧数据，只做增量迁移

## 存储容量

- IndexedDB上限：浏览器通常允许总容量的50-60%（Chrome无限制提示时更大）
- 纯文本日记：每篇约1-5KB
- 预估：1万篇日记 ≈ 10-50MB，完全在安全范围内
