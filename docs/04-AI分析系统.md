# AI分析系统

## 概述

使用DeepSeek API对用户日记进行多维度情绪和认知分析。所有分析手动触发，结果缓存到IndexedDB。未配置API Key时自动降级为模拟分析。

## 分析模型

| 配置项 | 值 |
|--------|-----|
| API | DeepSeek API |
| 模型 | deepseek-chat |
| 温度 | 0.7 |
| 最大Token | 2000 |
| 调用方式 | 服务端API Route（保护API Key） |

## VAD三维情绪模型

连续维度模型，比"正向/负向"更精确地描述情绪状态：

| 维度 | 范围 | 含义 | 示例 |
|------|------|------|------|
| Valence（愉悦度） | -1 ~ +1 | 情绪的正负面 | 狂喜+0.9 | 悲伤-0.7 | 平静0 |
| Arousal（唤醒度） | 0 ~ 1 | 情绪的激动程度 | 暴怒0.95 | 放松0.1 | 警觉0.6 |
| Dominance（支配度） | 0 ~ 1 | 对情绪的掌控感 | 自信0.8 | 无助0.1 | 纠结0.4 |

### VAD空间解读

| 区域 | VAD组合 | 典型情绪 |
|------|---------|---------|
| 高V + 高A + 高D | (+0.8, 0.8, 0.8) | 兴奋、自豪、有力量 |
| 高V + 低A + 高D | (+0.6, 0.2, 0.7) | 满足、安详、感恩 |
| 低V + 高A + 低D | (-0.7, 0.9, 0.2) | 愤怒、焦虑、恐惧 |
| 低V + 低A + 低D | (-0.6, 0.2, 0.1) | 抑郁、麻木、绝望 |
| 中V + 中A + 中D | (0, 0.3, 0.5) | 平静、清醒、自省 |

## Plutchik八情绪轮

基础情绪及其组合：

| 基础情绪 | 强度范围 | 与其它情绪的复合 |
|---------|---------|----------------|
| 喜悦 (Joy) | 宁静→喜悦→狂喜 | +信任=爱，+期待=乐观 |
| 信任 (Trust) | 接纳→信任→崇拜 | +喜悦=爱，+恐惧=屈从 |
| 恐惧 (Fear) | 担心→恐惧→恐怖 | +信任=屈从，+惊讶=畏惧 |
| 惊讶 (Surprise) | 分心→惊讶→震惊 | +喜悦=惊喜，+愤怒=愤怒 |
| 悲伤 (Sadness) | 忧郁→悲伤→悲痛 | +厌恶=悔恨，+愤怒=怨恨 |
| 厌恶 (Disgust) | 厌烦→厌恶→憎恶 | +悲伤=悔恨，+愤怒=鄙视 |
| 愤怒 (Anger) | 烦躁→愤怒→暴怒 | +厌恶=鄙视，+期待=攻击性 |
| 期待 (Anticipation) | 好奇→期待→警觉 | +喜悦=乐观，+恐惧=焦虑 |

每种情绪给0-1分，支持多标签（一篇日记可以同时有0.6的悲伤和0.3的期待）。

## 认知阶段

从认知行为疗法(CBT)和成长心理学提炼的五个阶段：

| 阶段 | 含义 | AI判断标准 |
|------|------|-----------|
| 觉察 (Awareness) | 开始注意到这个问题/情绪 | 文中有"我发现"、"我注意到"等 |
| 接纳 (Acceptance) | 接受现实，不抗拒 | 文中有"我接受"、"确实是这样"等 |
| 理解 (Understanding) | 开始理解为什么会这样 | 文中有"原来"、"因为"、"所以"等 |
| 重构 (Reframing) | 重新定义经历的意义 | 文中有"换个角度"、"其实是"、"反而"等 |
| 行动 (Action) | 准备好或已经做出改变 | 文中有"我决定"、"我开始"、"我做了"等 |

## System Prompt（核心）

```
你是一位专业的人生成长分析师，擅长从心理学角度分析个人记录。

你的任务是分析用户的日记内容，从以下维度给予反馈：

## 1. 情绪分析 (VAD三维模型)
- 愉悦度(Valence): -1到1
- 唤醒度(Arousal): 0到1
- 支配度(Dominance): 0到1

## 2. 情绪标签 (Plutchik八情绪)
对 joy/trust/fear/surprise/sadness/disgust/anger/anticipation 各打分0-1

## 3. 认知分析
- 认知阶段: 觉察/接纳/理解/重构/行动
- 情绪强度: 0到1
- 关联课题: 从[友情,亲情,爱情,工作,学习,情绪,认知,健身]中选择
- 建议标签: 2-4个更细粒度的标签

## 4. 反馈
- insight: 1-2句，点出隐藏的模式
- feedback: 2-3句，温暖而专业的回应
- followUpQuestion: 1个引导进一步思考的问题
```

## 模拟分析（降级方案）

当API Key未配置或API调用失败时使用。基于中文情感词典的关键词匹配：

- **正向词**：开心、高兴、快乐、满足、感恩、幸福、成功、进步、突破、理解、和解、释然
- **负向词**：难过、伤心、痛苦、焦虑、愤怒、生气、失望、孤独、迷茫、压力、害怕、后悔
- **平静词**：平静、淡定、接受、放下、顺其自然

模拟分析返回结果会标注"当前为模拟分析，配置API Key后可使用完整AI分析"。

## 分析结果缓存

- 分析结果存入 `analysisResults` 表
- 日记的 `hasAnalysis` 标记设为 true
- 再次查看同一日记时直接读缓存，不重复调用API
- **不自动更新分析**：即使用户修改了日记，也需手动重新分析（后续版本可优化）

## 多Agent扩展预留（v0.3+）

当前 `POST /api/analyze` 接口已预留 `agentType` 参数：

| agentType | 功能 | 触发周期 |
|-----------|------|---------|
| single（默认） | 单篇日记情绪分析 | 手动 |
| daily | 日间快速复盘 | 每日自动 |
| weekly | 周度情绪趋势报告 | 每周自动 |
| monthly | 月度课题进度总结 | 每月自动 |
| yearly | 年度人格特质分析 | 每年自动 |
| pattern | 跨课题模式识别 | 持续 |

v0.3+版本将实现这些Agent的不同System Prompt和调用策略。
