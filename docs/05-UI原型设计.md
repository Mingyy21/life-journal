# UI原型设计

## 设计原则

1. **极简**：减少一切不必要的UI元素，让写日记这件事本身成为焦点
2. **温暖**：暖色调 + 柔和圆角(12-16px) + 舒适间距(16-24px)
3. **渐进披露**：默认收起复杂信息（分析卡片默认折叠），用户主动展开
4. **移动优先**：先设计手机端(375-390px宽)，再适配桌面(max-w-2xl)
5. **平静感**：不使用鲜艳刺眼的颜色，整体是"翻开一本纸质日记"的感觉

## 响应式断点

| 断点 | 宽度 | 布局 |
|------|------|------|
| 手机 | < 640px | 单列，全宽，底部导航，16px内边距 |
| 平板 | 640-1024px | 单列，max-w-2xl居中，顶部导航 |
| 桌面 | > 1024px | 单列，max-w-2xl居中，更大字号可选 |

**策略**：不为平板/桌面写单独的复杂布局。始终单列阅读体验（像纸质日记），只是在不同屏幕上宽度不同。

## 微交互

| 交互 | 效果 |
|------|------|
| 点击写作入口 | 展开编辑器，动画滑入(slide-up 0.4s) |
| 选中课题按钮 | 填充课题颜色，文字变白，轻微缩放(scale 0.97) |
| 保存日记 | 按钮变"保存中..."，完成后编辑器滑出 |
| 分析中 | 加载动画旋转 + "AI正在分析你的日记..." |
| 展开分析卡片 | 内容区从0高度过渡到auto |
| 列表项悬停(桌面) | 卡片轻微阴影增强，箭头右移 |
| 列表项删除 | 左滑露出红色删除（移动端，v0.2） |
