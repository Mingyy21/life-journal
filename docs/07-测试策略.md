# 测试策略

## 测试分层

### 单元测试（Vitest）

**测试文件**：`src/__tests__/*.test.ts`

**数据库层测试**（`src/lib/db.ts`）：
- 数据库打开和初始化
- 预设课题正确写入（8个课题，分类正确）
- 日记CRUD：创建、读取、更新、删除
- 标签CRUD：创建、按课题筛选、删除
- 列表筛选：按课题ID、按标签ID、按关键词
- 分析缓存：保存分析→关联日记的hasAnalysis标记更新
- 课题计数：创建日记后topic.diaryCount+1

**AI分析测试**（`src/lib/ai.ts`）：
- 模拟分析返回有效VAD分数（在合法范围内）
- 模拟分析返回有效的八情绪标签（0-1范围）
- 正向文本→valence > 0
- 负向文本→valence < 0
- 平静文本→arousal < 0.5
- 分析结果JSON格式正确
- API Key未配置时返回模拟结果而非报错

**加密测试**（`src/lib/crypto.ts`，v0.2启用）：
- 加密再解密后原文一致
- 不同密码产生不同密文
- 错误密码无法解密
- PBKDF2密钥派生一致性

### 集成测试

- **AI分析全流程**：日记创建 → API Route调用 → 响应解析 → 结果存入analysisResults表 → 日记hasAnalysis标记 → 缓存读取
- **PWA注册**：Service Worker注册成功 → 静态资源缓存
- **IndexedDB性能**：1000条日记读写延迟测试

### E2E测试（Playwright）

**配置文件**：`playwright.config.ts`

**测试场景**：

#### 1. 完整用户流程
```
打开首页 → 点击写作入口 → 输入标题和内容 → 
选择"情绪"课题 → 点击保存 → 新日记出现在列表顶部 →
点击该日记 → 进入详情页 → 点击"AI分析" → 
等待分析完成 → 分析卡片展示 → 验证分析内容不为空
```

#### 2. 标签筛选流程
```
首页 → 点击"友情"课题筛选 → 列表仅显示友情日记 →
点击细化标签 → 进一步筛选 → 点击"全部" → 恢复全量
```

#### 3. 离线体验
```
首页正常加载 → 断开网络 → 写日记并保存 → 
日记成功保存到列表 → 查看历史日记正常 → 
恢复网络 → 数据仍在
```

#### 4. 移动端适配
| 设备 | 视口 | 关键验证点 |
|------|------|-----------|
| iPhone SE | 375×812 | 编辑器全宽、按钮大小≥44px |
| iPhone 14 Pro | 390×844 | 底部导航显示、安全区域 |
| iPad Mini | 768×1024 | 顶部导航(非底部)、内容居中max-w-2xl |
| Desktop Chrome | 1440×900 | 最大化阅读宽度、悬停效果 |

## CI/CD

### GitHub Actions 工作流

```yaml
name: CI
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npm run test
      - run: npx playwright test
```

### Vercel部署规则
- **main分支**：push自动构建 → 预览部署
- **生产发布**：手动Promote（或在GitHub中创建Release）
- **预览URL**：每个PR自动生成唯一预览链接

## 测试数据管理

- 使用`crypto.randomUUID()`生成测试用ID
- 每个测试前清空数据库（`db.delete()` → `db.open()`）
- E2E测试使用独立的测试数据库（`LifeJournalDB_test`）
- 不在生产IndexedDB上运行测试
