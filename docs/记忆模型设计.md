# 记忆模型设计（三层五机制）

> **状态**：设计完成，v0.3+ 实现
> **基础**：基于20篇认知科学文献和多个AI记忆系统研究总结

---

## 为什么不用Mem0？

Mem0的核心设计是"对话记忆"——"用户说过什么→提取事实→下次对话回忆"。但人生手记不是对话场景：

| | Mem0 | 人生手记需要 |
|---|---|---|
| 记忆单位 | "用户喜欢吃川菜" | "我和父亲吵架的过程+我的情绪+后来的反思" |
| 结构 | 扁平事实键值对 | 层级化：事件→感受→认知→成长 |
| 召回逻辑 | 语义相似即召回 | 频率权重+情绪强度+课题关联+时间衰减 |
| 遗忘 | 不重要就丢 | 不丢，低频的深埋，偶尔浮出 |
| 时间维度 | 无 | 纵向/横向/因果三链分析 |

## 三层记忆结构（借鉴Conway + Letta）

### L1 情节记录层（Episodic）
- **内容**：具体事件 + 情绪标注(VAD) + 当时的想法 + 与他人的对话
- **数量**：所有记录，永久保留
- **调取**：随频率自然衰减，低频的深埋但永远可检索
- **对应**：Conway模型中的"Episodic Detail"
- **例**："7月21日，和xx吵架，我感到愤怒(V:-0.7 A:0.8 D:0.3)，我说了..."

### L2 认知模式层（Semantic Frame）
- **内容**：从L1中提炼的语义框架(Barsalou Frame)，每个课题一个认知模型
- **数量**：每个课题一个Frame，持续更新
- **更新**：每次相关L1记录后微调
- **对应**：Conway的"General Event" + Barsalou的"Frame"
- **例**：{课题: 友情, 信任度: 0.6, 冲突模式: 回避型, 核心需求: 被理解, 典型情绪: 焦虑, 认知阶段: 理解}

### L3 生命叙事层（Self-Defining Memory）
- **内容**：改变自我认知的核心事件，构建"我是谁"的叙事
- **数量**：极少数（~20-50条），需AI判断后升级
- **更新**：季度/年，重大事件触发
- **对应**：Singer & Bluck的"Self-Defining Memory"
- **例**："我是一个习惯用退让来避免冲突的人"
- **判断标准**：高度情绪化 + 反复回忆 + 与核心人生课题相关 + 改变了你对自己的看法

## 五大核心机制

### 机制1：激活传播（借鉴ACT-R）

每条记忆的激活值：
```
a = B + Σ(S_i × W_i) + ε
```
- **B = 基础激活**：B = ln(n) - d × ln(t_since_last_access)
  - n = 被访问的总次数
  - d = 衰减率（高频课题d更低，低频课题d更高）
  - t = 距上次访问的时间
- **S_i × W_i = 扩散激活**：当前激活的课题/情绪节点会向关联记忆传播激活
  - 例：用户正在写"友情" → 所有topic=友情的记忆激活值临时+0.3
  - 例：当前情绪是"愤怒" → 所有Arousal>0.7的记忆激活值临时+0.2
- **ε = 噪声**：模拟人类记忆的不确定性

**激活阈值**：激活值超过阈值的记忆进入"工作记忆"（快速调取），低于阈值的保持"归档记忆"（深埋）。

### 机制2：情绪标记与记忆巩固（借鉴McGaugh + Tag&Capture）

- 情绪唤醒度(Arousal)>0.7 或 |Valence|>0.6 的记忆 → 自动标记为"需巩固"
- **巩固时间表**：第2天、第7天、第30天，AI主动生成回顾提示
- 巩固周期内，该记忆的**激活值不自然衰减**
- 每次回顾后，用户可"更新认知"（不是修改原始记录，而是追加新认知层）
- 睡眠中"重放"的模拟：AI在后台重新处理高情绪记忆，与其他记忆建立关联

### 机制3：去重与演化更新（借鉴Mem0）

```
新记录 → 向量嵌入 → 与已有L1记忆语义相似度比对
  │
  ├─ 相似度 > 0.85 → 追加到已有记忆的evolution链
  │   保留演化历史：v1(3个月前) → v2(2周前) → v3(今天)
  │   AI自动生成：认知变化摘要
  │
  └─ 相似度 ≤ 0.85 → 创建新记忆节点
```

**演化链的价值**：不只看"今天的你怎么想"，而是看到"你从A→B→C的认知路径"。

### 机制4：记忆优先级动态调整

```
priority = E × T × H × S
```
- **E (Emotion)**：情绪强度 0-1，VAD中Arousal值
- **T (Topic)**：课题关联度，关联的课题越多越重要
- **H (History)**：激活历史，过去被AI/用户调取的频率
- **S (Story)**：故事重要性 0-1，AI评估是否属于"自我定义记忆"
  - 用LLM判断："这条记录是否改变了你对自己的某种看法？"

高频课题 → H高 → priority高 → 保持在L2以上 → 快速调取
低频课题 → H低 → priority自然衰减 → 沉降到L1深层 → 偶尔浮出（契机触发）

### 机制5：跨周期审视（借鉴自传体推理）

**三个时间窗口的对比分析**：

#### 纵向链（同一课题的时间线）
```
你对"爱情"的理解：
  3月: "爱是无条件付出" (认知阶段: 觉察)
    ↓
  5月: "付出要有底线" (认知阶段: 理解)
    ↓
  7月: "爱是双向的经营" (认知阶段: 重构)
```
AI自动串起同一课题下不同时间的记录，输出认知演进报告。

#### 横向链（跨课题模式发现）
```
你的"回避冲突"模式出现在：
  → 友情（3次）：和朋友意见不合时选择沉默
  → 爱情（2次）：和伴侣吵架后冷战
  → 工作（4次）：对领导安排有意见但不说
  → 根源课题：自我 → 认知 → 冲突应对模式
```
AI发现你在不同课题下重复出现的思维/行为模式。

#### 因果链（事件→成长）
```
事件A(和父亲大吵) 
  → 情绪变化(愤怒→愧疚→理解)
  → 认知更新("他其实也是在用他的方式爱我")
  → 行为改变(主动打电话的频率增加了)
  → 课题进度(亲情: 35% → 52%)
```

## MemoryNode 数据模型（v0.3+）

```typescript
interface MemoryNode {
  id: string;
  type: "episodic" | "semantic_frame" | "self_defining";
  layer: 1 | 2 | 3;

  content: {
    raw: string;           // 原始记录/提炼内容
    summary: string;       // AI生成的摘要
    ai_insight: string;    // AI洞察
  };

  emotion: {
    valence: number;       // -1 ~ 1
    arousal: number;       // 0 ~ 1
    dominance: number;     // 0 ~ 1
    labels: string[];      // Plutchik情绪标签
    intensity: number;     // 综合强度 0-1
  };

  topics: string[];        // 关联课题ID

  activation: {
    base: number;          // 当前基础激活值
    last_access: Date;     // 上次被调取的时间
    access_count: number;  // 被调取总次数
    decay_rate: number;    // 自定义衰减率
  };

  consolidation: {
    needs_consolidation: boolean;  // 是否需要巩固
    review_schedule: Date[];       // 巩固时间表 [d2, d7, d30]
    last_reviewed: Date | null;    // 上次回顾时间
  };

  evolution: Array<{
    timestamp: Date;
    version_id: string;   // 指向之前版本的MemoryNode
    cognitive_delta: string; // AI生成的认知变化描述
  }>;

  graph_edges: Array<{
    target_id: string;
    relation: "triggered_by" | "caused_change" | "similar_to" | "contradicts" | "reinforces";
  }>;

  storytelling_weight: number; // 0-1，自我定义记忆权重
  created_at: Date;
  updated_at: Date;
}
```
