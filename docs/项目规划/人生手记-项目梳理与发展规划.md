## 一、第一性原理：这个项目最根本的作用

追问到底——人为什么要写日记？

| 层 | 问题 | 答案 |
|---|---|---|
| 表面 | 用户在做什么？ | 记录每天发生的事和感受 |
| 剥一层 | 为什么要记录？ | 怕忘记、想回顾、想倾诉 |
| 再剥 | 回顾带来什么？ | 看到自己的变化，获得某种确认感 |
| 再剥 | 确认感有什么用？ | 减少对自己的不确定和焦虑——人活着最底层的不安是"我不了解自己"和"我不知道该怎么走下去" |
| 本质 | **所以日记的根本作用** | **构建自我认知的连续性，在混乱中建立秩序** |

### 一句话定义

> **人生手记是一个"自我认知操作系统"——它帮你把零散的经历、情绪、思考结构化，让你看得清自己是谁、为什么这样、该往哪去。**

不是"记日记的工具"，而是"理解自己的引擎"。

### 三个核心命题

1. **觉察**（Awareness）：我今天经历了什么？感受如何？
2. **理解**（Understanding）：为什么我会这样反应？这背后是什么模式？
3. **方向**（Direction）：我想往哪里去？我正在成为什么样的人？

当前项目完成了"觉察"层，部分实现了"理解"层，"方向"层尚未触及。

### 3.2 数据流

```
AuthProvider → setUserId() → InitProvider → ensureDb()
                                              ↓
                                        db.open() (等待认证就绪)
                                              ↓
                                      initDefaultData() (默认领域+课题)
                                              ↓
                                      migrateFromDexie() (旧数据迁移，一次性)
                                              ↓
                                      ready = true → App 显示
                                              ↓
                                    各页面 useData hooks 发起 React Query
                                              ↓
                                     Supabase REST → 数据缓存 + UI 渲染
```

### 3.3 关键技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 状态管理 | React Query only | 简单够用，不需要 Zustand/Redux |
| 认证 | Supabase Email+Password | 个人使用，一个账号所有设备 |
| 表名 | 全小写 (lifedomains) | 匹配 Supabase REST 约定 |
| AI 分析 | 服务端 API route | 保护 API Key，客户端不可见 |
| 编辑器键盘 | visualViewport API | 比 resize 事件更准确 |
| 旧数据兼容 | Dexie 兼容层 | 无需改动 15+ 页面组件 |

## 四、已完成阶段详解

### 阶段一：数据架构重构（IndexedDB → Supabase 云同步）

**目标**：从浏览器 IndexedDB 迁移到云端，实现多设备数据互通。

**具体完成内容**：

- **Dexie-compat 适配层**：在 Dexie.js 和 Supabase 之间插入兼容代理，模拟 Dexie Table/Collection API → Supabase REST 翻译。核心文件 `dexie-compat.ts` 包含：
  - `SupabaseTable<T>`：模拟 Dexie.Table（get/add/put/update/delete/bulkAdd/toArray/where/orderBy/filter）
  - `SupabaseCollection<T>`：模拟 Dexie.Collection（equals/between/reverse/limit/offset/sortBy/toArray/first/last/count）
  - `supabaseProxy`：暴露所有表（lifeDomains/topics/diaries/analysisResults/events/insights/worldviews）
  - 自动 `userId` 注入 + `createdAt`/`updatedAt` 日期序列化/反序列化

- **AuthProvider**：基于 Supabase Auth 的认证上下文
  - 非阻塞初始化（页面不需要等待认证就绪再渲染）
  - `getSession()` 恢复持久化会话
  - `onAuthStateChange` 订阅认证状态变化
  - signIn/signUp/signOut 完整流程

- **InitProvider**：全局一次性初始化
  - authLoading 等待 → userId 就绪 → ensureDb() → 默认数据 → ready
  - 避免每个页面各自检查数据库是否就绪
  - 登出时 resetDbState

- **React Query 数据层**（useData.ts）：
  - queryKeys 工厂模式（类型安全的缓存键）
  - 引用数据 5 分钟 staleTime（topics/domains）
  - 日记数据 10 秒 staleTime（实时性要求高）
  - Mutations 自动 invalidateQueries
  - enabled: ready 模式（query 在初始化完成前不触发）

- **DataExportImport**：移除 Dexie 特有的 `db.transaction()`，改为顺序执行

### 阶段二：功能完善

**目标**：补齐核心日记应用应有的功能。

**具体完成内容**：

- **趋势分析服务端化**

- **编辑器键盘适配（移动端）**

  - 键盘弹出时保存栏从固定底部切换为流动布局

## 五、后续发展阶段

### 阶段四：AI 心理陪伴 ⭐ 最优先

**核心问题**：用户写了一段时间日记后，积累了情绪但无处释放。日记是单向的。AI 提供"回音"。

**4.1 AI 主动对话入口**

- 在课题详情页的"AI 干预"Tab 中嵌入对话界面
- 对话界面：消息气泡列表 + 底部输入框（类似微信/iMessage）
- 用户点开即可与 AI 讨论当前课题的感受
- AI 需要基于该课题下的所有日记内容做上下文理解（不是通用闲聊）

**4.2 情绪累积阈值检测**

- 在 useIntervention hook 中实现检测逻辑
- 定期扫描某课题最近 N 篇 diary 的 analysisResult
- 如果连续多篇 Valence 偏低（VAD 模型情绪值 < 阈值），标记为"需关注"
- 用户进入该课题时弹出："你最近在 [课题名] 上似乎积累了压力……"
- 阈值可配置，避免过度打扰

**4.3 CBT 风格认知重构对话**

AI 的回应策略借鉴认知行为疗法，三层递进：

| 层级 | 目标 | 示例回应 |
|------|------|----------|
| 共情 | 让用户感到被听见 | "听起来这件事让你很沮丧，换作是我可能也会有同样的感受。" |
| 引导 | 帮用户看到不同的角度 | "你提到'我总是搞砸'——有没有哪一次其实结果还不错的？" |
| 重构 | 帮用户建立新认知 | "如果用一个更温和的说法来描述这件事，你会怎么说？" |

**4.4 对话历史持久化**

- 新建表 `therapy_sessions`：

```sql
CREATE TABLE therapy_sessions (
  id TEXT PRIMARY KEY,
  "userId" UUID NOT NULL DEFAULT auth.uid(),
  "topicId" TEXT NOT NULL,
  messages JSONB NOT NULL DEFAULT '[]',
  summary TEXT,
  "createdAt" TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  "updatedAt" TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

- 对话摘要由 AI 在对话结束后自动生成
- 用户可以回顾历史对话

**4.5 技术方案**

```
对话请求 → POST /api/therapy { topicId, message }
                   ↓
          读取该课题最近日记 + 对话历史
                   ↓
          构建 Prompt（含日记上下文 + CBT 指导）
                   ↓
          调用 DeepSeek API
                   ↓
          存储对话（新消息追加到 messages JSONB）
                   ↓
          返回 AI 回复文本
```

### 阶段五：Worldviews 三观模块

**核心理念**：日记不仅是记录"发生了什么"，更重要的是提炼出"我相信什么"。这个模块帮助用户从日常记录中挖掘出自己的信念体系。

**5.1 世界观自动提炼**

- 从 Insights（感悟）中做聚类分析
- 例如：用户反复写"我觉得自己不够好"、"我又搞砸了" → 聚类为"自我价值感不足"
- 每次新 Insight 产生时，AI 分析是否匹配已有世界观
- 新建表 `worldviews` 已预留

**5.2 世界观图谱可视化**

- 节点图展示信念体系（类似知识图谱）
- 每个节点 = 一个世界观
- 节点大小 = 影响强度（关联的日记/感悟数量）
- 连线 = 信念之间的关联（"自我价值感不足" → "害怕被否定"）
- 时间轴动画：哪些信念变强、哪些松动

**5.3 信念矛盾温和提示**

- AI 识别日记中的逻辑矛盾
- 场景：周一写"我要学会接纳不完美的自己"，周三写"我这次表现太差劲了"
- AI 提示："你注意到了吗？上次你说想过要接纳自己，但这次对自己很严厉。这没关系，只是值得留意。"

**5.4 世界观清单**

- 类似于一个"个人说明书"
- 用户可以主动编辑/确认/删除自动提炼的世界观
- 最终形成一份属于用户自己的人生哲学笔记

### 阶段六：趋势与模式深度分析

**当前基础**：基本统计（日记数量、情感分布）、DeepSeek 生成趋势报告。

**6.1 模式自动发现**

- 时间模式："每次周末你心情都更好"、"周三比较低落"
- 触发因素："每当提到[某话题]，你的焦虑值会升高"
- 关联模式："健身日 → 情绪分 +20%"
- 通过分析日记 + analysisResult 的关联数据

**6.2 周报/月报自动生成**

- AI 每周/月生成个人成长报告
- 内容：情绪走势、关键事件、认知变化、下周关注点
- 在首页展示卡片式摘要
- 可推送通知

**6.3 多维度交叉分析**

| 维度 | 问题 |
|------|------|
| 课题 × 情绪 | 哪个课题让你最焦虑？哪个让你最放松？ |
| 时间 × 效率 | 早上写 VS 晚上写，情绪有什么不同？ |
| 事件 × 时间 | 未解决事件平均拖延多久？ |
| 认知阶段 × 趋势 | 你是在进步还是在循环？ |

| **VAD 情感模型** | Valence（愉悦度）/ Arousal（激活度）/ Dominance（支配度）。当前 AI 分析已在用。 |
| **Kegan 成人发展理论** | 人的认知发展分为不同阶段。长期可使用户评估自己在哪个阶段。 |
