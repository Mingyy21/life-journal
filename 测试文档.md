## 测试执行规则

1. **分阶段执行**：按阶段一 → 阶段二 → 阶段三的顺序执行
2. **阶段门禁**：每个阶段完成后执行对应测试清单，**全部通过**后才能进入下一阶段
3. **重试上限**：每个阶段测试最多重试 3 次，超过则回退到上个 commit 重新分析
4. **构建验证**：每个阶段开始前 `npm run build` 必须通过
5. **回归**：阶段三完成后执行完整回归测试（第 2-14 节）

### 未解决的问题 / 已知限制

1. **邮箱确认**：如果 Supabase 开启了邮箱确认功能，注册后需到邮箱点击确认链接。建议关闭（Authentication → Settings → Disable "Confirm email"）。
2. **重定向 URL**：Supabase 的 Site URL 需要设置为 `https://lifjourney.netlify.app`，否则密码重置等邮件中的链接指向 `localhost:3000`。
3. **AI 分析配置**：需要配置 `DEEPSEEK_API_KEY` 环境变量才能使用 AI 分析功能。未配置时自动降级为本地规则分析。
4. **离线不可用**：由于使用 Supabase 云数据库，断网时无法读写数据（相比 IndexedDB 版本的离线能力下降）。
5. **数据迁移**：从旧版（IndexedDB）升级的用户需要先在旧版导出 JSON，登录新版后导入。
