# 事件创建与双向链接 — 2025 研究综述

> 采集时间：2026-07-21
> 关键词：event creation UX, bidirectional linking, GTD inbox processing, note-to-task conversion

---

## 1. 五种巧妙的"从笔记创建事件/任务"的UX模式

### 方式1：笔记内一键提升 (Note → Event in one tap)
**来源**：AI Notes Voice (2025), GTD wizard pattern

写笔记时，如果觉得"这不止是一篇笔记"，点一个按钮，当前笔记变成新事件的第一条记录。**捕获和决策分离**——不打断写作流。

→ 人生手记应用：写日记 → 写完点"升级为事件" → 输入事件名 → 自动关联。

### 方式2：收件箱批量整理 (Inbox Triage)
**来源**：Mindwtr, GTD Flow plugin, Obsidian inbox processing

在整理视图中显示未归类的条目，用户多选后归入已有项目或创建新项目。GTD核心原则：**先把所有东西扔进收件箱，然后定期整理**。

→ 限制策略：只显示当日+近3天的未关联日记，避免积压焦虑。

### 方式3：从项目/课题页直接创建 (Project-first creation)
**来源**：cc-toolkit, GTD next-action pattern

不需要先有日记——先定义事件框架，然后每次写日记时选择归属。适合"持续事件"。

### 方式4：系统温和提示 (Contextual suggestion)
**来源**：Sentiment-driven adaptive UI (DearDiary 2025), AI auto-classification (Obsidian + Gemini)

当系统检测到连续N篇日记涉及相同课题/关键词时，以极轻的方式提示创建事件。**不主动打扰，在用户可能需要时恰好出现**。

### 方式5：写作时下拉选择 (Inline selection)
**来源**：Quick-add syntax pattern (Mindwtr), unified daily dashboard (ADHD_Bujo 2025)

日记编辑器中直接选择"所属事件"，最简单直接的关联方式。选"无"就是日常日记，选已有事件就关联上。

## 2. 双向链接的UX模式

**来源**：Roam Research, Obsidian, Logseq

### 核心概念
双向链接：Note A链接到Note B时，自动在Note B看到来自Note A的反向链接。创建**网络化思维**，而非刚性文件夹层级。

### 关键设计原则

| 原则 | 说明 |
|------|------|
| **自动反向链接** | 关联是双向的——A关联B则B自动显示关联到A |
| **链接上下文** | 不仅显示"被链接了"，还显示链接来源的上下文片段 |
| **非超链接** | 不是网页跳转，是数据层的关联关系 |
| **图视图（可选）** | 可视化所有关联网络，但不是必须的 |

### Roam vs Obsidian 的核心差异

| 维度 | Roam Research | Obsidian |
|------|---------------|----------|
| 存储 | 云端 | 本地Markdown文件 |
| 链接粒度 | 块级（段落） | 文件级 |
| 价格 | $15/月 | 免费（个人使用） |
| AI | 第三方/社区 | 丰富的插件生态 |

→ 人生手记的选择：Dexie.js本地存储 + 数据层关联（非超链接）+ 未来可选图视图。

## 3. GTD收件箱处理模式

**来源**：GTD (David Allen), Mindwtr, Flow plugin

### 顺序处理向导（一次一个问题）

```
条目 → "可行动吗？"
   → 否 → "是什么？" → 删除 / 将来也许 / 参考资料
   → 是 → "少于2分钟？" → 立刻做
          → "谁来做？" → 我来 / 委托
             → "属于哪个项目？"（可选）
```

**核心UX洞察**：一次只问一个问题，降低认知负荷。Flow Obsidian插件以内联模态框形式展示，不丢失上下文。

### AI作为"保守的记账员"
**来源**：getkeppt.com, dev.to

用户口头输入，AI移动文件。核心设计原则：**不做创意解释、不静默编辑、不"好心"重组**——只展示diff，不确定时询问。信任是核心UX指标。

### 快速添加语法（内联上下文）
**来源**：Mindwtr

单行捕获含嵌入元数据：
```
Call plumber @phone @home
Research topic #focused +WorkProject
```

→ 适应后，高级用户可绕过处理向导直接分类，普通用户继续用向导。

## 4. 对人生手记的设计启示

| 研究来源 | 启示 |
|----------|------|
| AI Notes Voice | 日记内"升级为事件"按钮，捕获与决策分离 |
| DearDiary 2025 | 温和提示，不打扰用户 |
| GTD Flow | 批量整理限制3天内，避免积压 |
| Roam/Obsidian | 数据层双向关联，非超链接 |
| Mindwtr quick-add | 日记编辑器中下拉选择所属事件 |
| ADHD_Bujo | 统一视图：日记+事件+感悟共存 |
